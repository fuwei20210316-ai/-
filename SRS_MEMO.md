# 间隔重复系统 (SRS) 逻辑设计备忘录

**日期**: 2026-03-30
**版本**: 1.0
**核心算法**: SM-2 (SuperMemo-2)

## 1. 核心参数定义

系统通过以下四个关键参数来追踪每道错题的记忆状态：

| 参数 | 名称 | 说明 |
| :--- | :--- | :--- |
| `q` | 质量 (Quality) | 用户回答的质量评分（0-5分）。在本项目中，正确为 5 分，错误为 0 分。 |
| `n` | 重复次数 (Repetition) | 连续正确回答的次数。 |
| `ef` | 简易度因子 (Easiness Factor) | 衡量题目难易程度的系数，初始值为 2.5。 |
| `i` | 间隔天数 (Interval) | 距离下一次复习的天数。 |

## 2. 计算逻辑 (SM-2 算法)

每次复习后，系统会根据回答质量 `q` 更新参数：

### 2.1 间隔天数 `i` 的更新
- **如果 `q >= 3` (回答正确)**:
  - 如果是第 1 次正确 (`n = 0`): `i = 1` 天
  - 如果是第 2 次正确 (`n = 1`): `i = 6` 天
  - 如果是第 3 次及以上正确 (`n > 1`): `i = round(上一次 i * ef)`
  - `n = n + 1`
- **如果 `q < 3` (回答错误)**:
  - `i = 1` 天 (重置间隔)
  - `n = 0` (重置重复次数)

### 2.2 简易度因子 `ef` 的更新
公式: `nextEf = ef + (0.1 - (5 - q) * (0.08 + (5 - q) * 0.02))`
- `ef` 的最小值为 1.3。即使题目再难，间隔增长率也不会低于 1.3。

### 2.3 下次复习日期
`nextReviewDate = 当前时间 + (i * 24 * 60 * 60 * 1000) 毫秒`

## 3. 数据库结构 (WrongQuestion)

错题记录存储在 `wrongQuestions` 集合中，包含以下 SRS 相关字段：

```typescript
interface WrongQuestion {
  // ... 题目内容字段
  repetition: number;    // 对应算法中的 n
  efactor: number;       // 对应算法中的 ef
  interval: number;      // 对应算法中的 i
  nextReviewDate: number;// 下次复习的时间戳
  wrongCount: number;    // 累计错误总次数
  status: 'active' | 'mastered'; // 状态：激活中 或 已掌握
}
```

## 4. 业务流程集成

### 4.1 初始入库 (常规测试)
- 当用户在常规测试中答错题时，调用 `addWrongQuestion`。
- **初始值**: `ef = 2.5`, `n = 0`, `i = 1`, `status = 'active'`。
- **去重逻辑**: 如果该题已存在，则仅增加 `wrongCount` 并将状态设回 `active`。

### 4.2 复习更新 (错题重测)
- 当用户在错题集进行重测后，调用 `updateWrongQuestionSRS`。
- **评分映射**: 
  - 正确 -> `q = 5`
  - 错误 -> `q = 0`
- **掌握判定**: 当 `repetition >= 5` 时，`status` 自动变为 `'mastered'`，该题将不再出现在待复习列表中。

## 5. UI 交互逻辑

- **全局提醒**: 首页检测 `nextReviewDate <= Date.now()` 的 `active` 题目数量。
- **书籍维度**: 在书籍详情页汇总该书下的到期题目。
- **复习入口**: 提供“立即复习”按钮，筛选出所有到期题目组成专项测试。

---
**备注**: 本逻辑旨在通过科学的间隔安排，帮助用户以最少的复习次数达到长久记忆的效果。后期可根据用户反馈调整 `mastered` 的判定阈值（当前为 5 次）。
