# 书架与题库功能需求备忘录

## 1. 核心流程
1. **搜索**：用户搜索目标书籍。
2. **预览**：点击书籍查看详情预览，确认目标。
3. **收藏**：将书籍收藏至“书架”。
4. **精读计划**：在书架中点击书籍，设置精读测试计划。
5. **测试**：根据计划进行章节测试。

## 2. 功能特性与 VIP 权限控制

| 功能 | 免费用户 | VIP 用户 |
| :--- | :--- | :--- |
| **测试题量** | 5 题 | 5 / 10 / 15 题 |
| **语言支持** | 仅汉语 | 支持多语言 |
| **应试真题强化** | 不支持 | 支持 |
| **题型选择** | 仅单选题 | 单选、多选、填空、判断、主观表述题 |
| **计时功能** | 支持 (倒计时/正计时) | 支持 (倒计时/正计时) |

## 3. 关键业务逻辑
* **计划生效范围**：打卡计划一旦设置，作用于全书所有章节的答题规则。
* **计划灵活性**：支持用户随时中途更改打卡计划。
* **测试范围限制**：单次测试最大范围为 3 个章节。
* **时间提醒**：书架内提供“今日待测”的应用内 UI 提醒。

## 4. 数据结构需求 (扩展)
* **UserBook**：
    * `folderId`: string (自定义文件夹)
    * `mastery`: number (综合掌握度)
    * `planConfig`: object (每日目标、开始日期、题量、题型、计时模式等)
    * `lastActiveAt`: timestamp
    * `roadmapData`: array (章节状态)
* **UserProfile**：
    * `isVip`: boolean (权限控制)
